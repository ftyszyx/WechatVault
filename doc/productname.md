# 产品命名分析

## 背景

项目当前使用 `WechatVault` 作为名称。产品用途是读取并归档用户本人 Windows 电脑上已经保存的本地聊天数据，当前技术验证对象为 Windows 微信客户端。

名称中的 `Wechat` 容易让用户产生以下误解：

- 误认为产品由微信或腾讯官方提供、授权或背书。
- 误认为产品仅服务于微信，限制未来支持其他本地聊天数据源的空间。
- 在产品图标、安装包、搜索结果和推广文案中增加商标及品牌混淆风险。
- 用户可能将工具的安全性、数据完整性和售后责任错误地归因于微信官方。

因此，产品品牌应使用不包含 `WeChat`、`微信`、`Weixin` 或 `wx` 的中性名称。微信相关名称只用于必要的兼容性说明和技术文档。

## 推荐名称

### LocalMsgVault

推荐将产品名称定为：

- 英文产品名：`LocalMsgVault`
- 中文产品名：`本地消息库`
- GitHub 仓库名：`local-msg-vault`
- Rust package：`local-msg-vault`
- PoC 可执行文件：`local-msg-vault-poc.exe`
- 产品描述：`个人本地聊天数据归档工具`

该名称的优点：

- `Local` 明确强调数据在本机处理，有助于建立隐私预期。
- `Msg` 表明处理对象是消息，但不绑定特定平台。
- `Vault` 延续现有项目的归档和保管含义，迁移成本较低。
- 名称可以覆盖联系人、会话、附件和未来新增的数据源。

该名称的不足：

- 英文名称偏工具化，中文传播时需要搭配“本地消息库”。
- `Vault` 和 `Msg` 都是常用词，正式发布前仍需检查同名软件、商标、域名和软件包占用情况。

当前推荐属于产品命名建议，不构成商标或法律核验结论。

## 备选名称

| 名称 | 优点 | 不足 |
| --- | --- | --- |
| `MsgVault` | 简短，容易从旧名称迁移 | 未直接强调本地处理，同名概率较高 |
| `LocalChatArchive` | 含义直接，用户容易理解 | 较长，品牌辨识度一般 |
| `ChatLedger` | 有记录、索引和可追溯含义 | 容易让人联想到审计或企业合规产品 |
| `ConversationVault` | 表意完整，可覆盖多种会话 | 名称较长，不适合命令行和包名 |
| `聊档` | 中文简短，突出聊天归档 | 英文仓库名和海外传播需要另行设计 |

综合产品定位、迁移成本和未来扩展性，首选仍为 `LocalMsgVault`。

## 品牌与兼容性表述

品牌名称和兼容性说明应严格分开。

推荐表述：

> LocalMsgVault 是一款个人本地聊天数据归档工具。当前版本适配 Windows 微信 4.1.11.24，仅处理用户本人电脑上已经保存的数据。

不推荐表述：

- 微信聊天记录官方导出工具
- 微信数据保险箱
- 微信授权备份工具
- WeChat Exporter
- 微信一键恢复

产品首页、安装程序和仓库 README 应展示以下声明：

> 本项目为非官方工具，与腾讯及微信无关联。产品仅处理用户主动授权访问的本地数据。

技术文档可以继续使用 `Weixin.exe`、`Weixin.dll`、`xwechat_files` 和具体版本号。这些名称用于准确描述兼容对象，不作为产品品牌或来源暗示。

## 视觉与传播规则

- 不使用微信或腾讯的 Logo、绿色聊天气泡、官方字体风格及近似图标。
- 不使用“官方”“授权”“合作”“认证”等可能暗示关联关系的词语。
- 主视觉应体现本地、归档、搜索和隐私，而不是模仿微信客户端。
- 搜索标题以产品名和功能为主，例如“LocalMsgVault - 本地聊天数据归档工具”。
- 在兼容性页面单独列出支持的平台、客户端版本和已验证数据类型。
- 不承诺导出云端数据、恢复已删除数据或绕过账号登录。

## 代码与仓库改名范围

确定名称后，需要统一修改以下位置：

1. GitHub 仓库名称：`WechatVault` 改为 `local-msg-vault`。
2. `Cargo.toml`：package name 改为 `local-msg-vault`。
3. CLI 输出：将 `WechatVault feasibility PoC` 改为 `LocalMsgVault feasibility PoC`。
4. README：修改标题、介绍、编译命令和可执行文件名。
5. 安装包：修改应用标题、发布产物名称、卸载项名称和签名描述。
6. UI：修改窗口标题、关于页面、错误报告和日志中的产品名称。
7. 网站及文档：统一产品名、中文名、免责声明和兼容性描述。
8. Git remote：仓库重命名后更新本地 `origin` 地址。

内部 Windows 适配代码不需要重命名 `Weixin.exe` 等客户端技术标识。

## 发布前检查

正式采用名称前应完成以下检查：

- 检索 GitHub、常见软件商店和搜索引擎中的同名项目。
- 检索目标市场的相关商标类别。
- 检查主要域名、社交账号和软件包名称是否可用。
- 检查名称在中文和英文语境中是否存在负面或歧义含义。
- 确认安装包、图标和宣传材料不会造成官方关联误解。
- 在隐私政策和用户协议中明确本地处理范围与用户授权边界。

## 当前结论

`WechatVault` 不适合作为正式产品名。建议使用 `LocalMsgVault`，中文名为“本地消息库”，并将“支持 Windows 微信特定版本”作为兼容性说明，而不是品牌的一部分。
